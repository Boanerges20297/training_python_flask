# Plan: 01-02 — Dashboard & Component Polish

## Goal
Apply the professional design system to the Dashboard and standard components. This involves refactoring the Dashboard's Bento grid styles to be cleaner and more focused on data, and injecting styles to normalize SweetAlert popups.

## Context
The Dashboard is currently "escandaloso" (loud) with too many animations and glows. We need to streamline it to support financial decision-making. SweetAlert also needs to be brought in line with the new 12px max radius.

## Wave 1: Dashboard Styles Refactor
- [ ] **Task 01-02-01**: Refactor `DashboardView.module.css`.
    - **Action**: 
        - Update all `border-radius: 1.75rem` to `var(--radius-xl)` (which is now 0.75rem).
        - Remove `radial-gradient` backgrounds and `box-shadow` glows (L36, L43, L94).
        - Simplify hover effects (L98-L124): remove scaling/transform and stick to subtle border-color changes or slight shadow depth.
        - Set `backdrop-filter` to a lower value (e.g., `blur(8px)` instead of `20px`).
    - **Read First**: `frontend/src/modules/admin/views/DashboardView.module.css`
    - **Acceptance Criteria**: 
        - `border-radius` values are mapped to variables or reduced to `0.75rem`.
        - `radial-gradient` removed from `.loadingState`.
        - Hover transforms (`scale(1.01)`) are removed.

- [ ] **Task 01-02-02**: Clean up Dashboard Content (Charts and Cards).
    - **Action**: 
        - In `DashboardView.tsx`, ensure Recharts colors use the new professional palette.
        - Remove redundant taglines if they add noise.
    - **Read First**: `frontend/src/modules/admin/views/DashboardView.tsx`
    - **Acceptance Criteria**: 
        - Chart colors are consistent with tokens.

## Wave 2: Component Standardization
- [ ] **Task 01-02-03**: Normalize SweetAlert (Swalert) Styles.
    - **Action**: 
        - Add global overrides in `index.css` (or a dedicated `sweetalert.css`) to force Swalert popups to use `var(--radius-xl)` (0.75rem) and the project's primary button styles.
        - Target `.swal2-popup`, `.swal2-styled.swal2-confirm`, etc.
    - **Read First**: `frontend/src/index.css`
    - **Acceptance Criteria**: 
        - Global CSS contains overrides for `.swal2-popup` setting `border-radius` to `var(--radius-xl)`.
