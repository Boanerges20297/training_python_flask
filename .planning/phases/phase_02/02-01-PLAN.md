# Plan: 02-01 — Security Infrastructure

## Goal
Establish the database model and configuration for persistent JWT management.

## Context
We are moving from a mock in-memory blocklist to a database-backed one to ensure logouts are persistent. We also need to configure strict claim validation.

## Wave 1: Database Model
- [ ] **Task 02-01-01**: Create `TokenBlocklist` model.
    - **Action**: Create `backend/app/models/token_blocklist.py` with fields: `id`, `jti`, `type`, `user_id`, `created_at`.
    - **Acceptance Criteria**: `TokenBlocklist` is defined and imported in `app/models/__init__.py`.

## Wave 2: Configuration Update
- [ ] **Task 02-01-02**: Update JWT Configuration.
    - **Action**: 
        - Set `JWT_ACCESS_TOKEN_EXPIRES = 3600` (60 min).
        - Ensure `JWT_DECODE_ISSUER` and `JWT_DECODE_AUDIENCE` are strictly checked (via `JWT_IDENTITY_CLAIM` or explicit settings if needed).
    - **Read First**: `backend/config.py`
    - **Acceptance Criteria**: `config.py` reflects the new expiry and strict validation settings.
