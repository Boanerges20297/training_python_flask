# Plan: 03-01 — Mock Data Seeding

## Goal
Create a robust seeding script to populate the database with realistic business data.

## Context
The user needs realistic prices and volume to validate the Dashboard. We'll use the `Faker` library to generate Portuguese (Brazil) data.

## Wave 1: Script Preparation
- [ ] **Task 03-01-01**: Create `backend/scripts/seed_data.py`.
    - **Action**: 
        - Initialize `Faker('pt_BR')`.
        - Define a `seed()` function that:
            1. Clears current tables (Admin, Cliente, Barbeiro, Servico, Agendamento).
            2. Creates 1 Admin.
            3. Creates 5 standard professional Services with realistic prices.
            4. Creates 5 Barbers and associates them with services.
            5. Creates 200 Clients.
            6. Creates 1000+ Appointments over the last 90 days.
    - **Acceptance Criteria**: Script runs without errors and populates all tables.

## Wave 2: Business Logic Injection
- [ ] **Task 03-01-02**: Ensure realistic distribution.
    - **Action**: 
        - Bias appointment times towards weekends and peak hours (10h-12h, 16h-19h).
        - Randomly assign statuses: 80% Concluído, 10% Cancelado, 10% Pendente/Futuro.
    - **Acceptance Criteria**: Dashboard "Ondas de Demanda" shows peaks and valleys consistent with a real barber shop.

---

## Verification
- [ ] Run `python backend/scripts/seed_data.py`.
- [ ] Check record counts in the DB (Admin, Barbers, etc.).
- [ ] Verify Dashboard visualizations reflect the new data.

## must_haves
- Script uses `Faker`.
- Prices are realistic (e.g., Corte > R$ 40).
- 90 days of history.
